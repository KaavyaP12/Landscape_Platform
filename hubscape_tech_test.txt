# Backend Developer Technical Test - Landscape Gardening Platform

## Overview
This technical test is designed to evaluate a backend developer's ability to work with Django monoliths, design microservice architectures, implement serverless solutions, and work with AWS services including SNS, SQS, and Lambda in the context of a landscape gardening and scheduling platform.

## Test Duration
**2-3 hours** (can be completed over multiple sessions)

---

## Part 1: Django Monolith Maintenance (45 minutes)

### Scenario
You're maintaining a Django landscape gardening platform with the following models:

```python
# models.py
from django.db import models
from django.contrib.auth.models import User

class Client(models.Model):
    name = models.CharField(max_length=200)
    email = models.EmailField()
    phone = models.CharField(max_length=20)
    address = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

class Service(models.Model):
    name = models.CharField(max_length=200)
    description = models.TextField()
    base_price = models.DecimalField(max_digits=10, decimal_places=2)
    duration_hours = models.IntegerField()
    is_active = models.BooleanField(default=True)

class Appointment(models.Model):
    client = models.ForeignKey(Client, on_delete=models.CASCADE)
    service = models.ForeignKey(Service, on_delete=models.CASCADE)
    scheduled_date = models.DateTimeField()
    status = models.CharField(max_length=20, choices=[
        ('scheduled', 'Scheduled'),
        ('in_progress', 'In Progress'),
        ('completed', 'Completed'),
        ('cancelled', 'Cancelled')
    ])
    notes = models.TextField(blank=True)
    created_at = models.DateTimeField(auto_now_add=True)

class TeamMember(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    specialties = models.ManyToManyField(Service)
    is_available = models.BooleanField(default=True)
```

### Tasks

1. **Performance Optimization** (15 minutes)
   - Optimize the following view to reduce database queries:

```python
def appointment_list_view(request):
    appointments = Appointment.objects.all()
    for appointment in appointments:
        print(f"Appointment {appointment.id} for {appointment.client.name}")
        print(f"  - Service: {appointment.service.name}")
        print(f"  - Scheduled: {appointment.scheduled_date}")
    return JsonResponse({'appointments': list(appointments.values())})
```

2. **Database Migration** (15 minutes)
   - Create a migration to add a `priority` field to the Service model (choices: 'low', 'medium', 'high')
   - Write a data migration to set existing services to 'medium' priority

3. **API Development** (15 minutes)
   - Create a REST API endpoint using Django REST Framework that:
     - Lists appointments with pagination
     - Allows filtering by date range and status
     - Includes proper error handling and validation
     - Implements rate limiting (50 requests per minute per user)

## Part 4: SNS->SQS Message Queues with Lambda Background Workers (30 minutes)

### Scenario
Implement a background job processing system using SNS->SQS->Lambda pattern for handling:
- Appointment scheduling workflows
- Weather check notifications
- Client reminders
- Team member notifications

### Tasks

1. **SNS Topic and SQS Queue Setup** (10 minutes)
   - Design SNS topics for different event types:
     - `appointment-created`
     - `weather-alert`
     - `reminder-due`
     - `team-notification`
   - Create corresponding SQS queues with appropriate configurations

2. **Lambda Background Workers** (20 minutes)
   - Implement Lambda functions that process messages from SQS queues:
     - `appointment-processor`: Processes new appointments
     - `weather-processor`: Checks weather and sends alerts
     - `reminder-processor`: Sends client reminders
     - `notification-processor`: Sends team notifications
   - Include proper error handling and retry logic
